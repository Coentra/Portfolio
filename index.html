Agility Registration Website

Case Study: Agivent, a Dog Agility Competition Platform (Norway)

(Client name changed for confidentiality)




Problem

Dog agility enthusiasts in Norway struggle with:

  • No centralized platform for competitions.
  • Difficulty registering dogs and finding event details.
  • Lack of visibility into start numbers, schedules, and live results.
  • Frustrating user experience that discourages participation.
  • Example issue: Many users quit halfway through registration because the process is too complex and slow.

Solution

Design and prototype a dedicated website that:

  • Simplifies competition search & registration.
  • Provides live results, start numbers, and class schedules.
  • Offers personalized overview.
  • Encourages more people to join and stay engaged in the agility community.

Project Overview

  • Type: Exam Project, Noroff School of Technology and Digital Media
  • Role: UX Designer & UI Designer
  • Duration: 5 weeks
  • Tools: Figma, Miro, Adobe Illustrator, Google Docs/Sheets

Design Process

1. Empathize


I started with 5W + H questions to frame research goals.

High priority questions included:

  • Why do users quit during registration?
  • Why are people not attending competitions?
  • What do users think of the current system?

Why this method?

The 5W+H framework helped me define clear research goals and ensure I wasn’t just gathering data, but focusing on actionable insights.


Research Methods & Key Insights

  • Observation (naturalistic): Shadowed a participant using the current system → identified hidden frustrations (slow loading, endless errors).
  • Competitive research: Studied systems in US, Sweden, Denmark, Germany, Finland.
    • Only 1 country offered live results.
    • US had the strongest system, but lacked features like starting order.
  • User interviews (6 participants):
    • Users quit midway through registration..
    • New users can’t easily find competitions.
    • Current system feels outdated and slow.

2. Define


I consolidated findings with Affinity Mapping.

Key problems:

  • Information scattered across multiple sources.
  • Registration crashes during high-traffic events.
  • Users stressed because they don’t know when their class starts.

Opportunity:

Create a single, reliable platform for registration, schedules, live results, and event information—all in one place.



3. Personas


Meet Elise, 32 – my primary persona.

Key problems:

  • Goals: Train her dog to Class 3, plan competitions, and easily access live results.
  • Frustrations: Stress from unclear schedules, difficulty finding information, and no live result updates.

The persona ensured I kept user goals, not just system features, at the center of design decisions.



4. Ideation


I ran a “How Might We” brainstorming session, generating solutions around:

  • Registration flow.
  • Live result display.
  • Class start order and schedules.
  • Easy competition discovery.

Outcome:

A website where users can:

  • Register dogs and enter competitions.
  • View live results.
  • See upcoming events and start order.
  • Access all competition info in one place.
  • Enable clubs to host competitions seamlessly.

5. Information Architecture & User Flows


I structured the platform around three key user flows:

  1. Register a dog.
  2. Enter a competition.
  3. Place a run for sale.

Organizing this upfront clarified how users would navigate the system without getting lost.



6. Prototyping & Testing


Mid-fidelity prototypes

  • Created both desktop and mobile versions for early usability testing.

Usability Testing (5 users)

Tasks tested:

  • Register a dog.
  • Register for a competition.
  • Place a run for sale.

Findings:

  • Users navigated easily, but improvements were needed:
    • Add
      filters for events.
    • Improve homepage layout.
    • Show ongoing events.
    • Highlight registration open/close dates more clearly.

Why usability testing?

Even though this was a student project, I wanted evidence-based design decisions. Testing confirmed my assumptions and highlighted new opportunities.

Iterations

  • Implemented homepage redesign.
  • Added filters.
  • Improved visibility of registration timelines.

High-fidelity Prototype & Design System

  • Built out polished UI with consistent colors, typography, and components for scalability.

Results & Impact


The redesigned platform solved the key user frustrations:

  • Users could register without crashes.
  • Clear visibility of start times & numbers reduced stress.
  • Live results increased excitement and engagement.
  • Event organizers gained a reliable, modern tool.

What’s Next?


Future improvements:

  • Add a competition organizer dashboard.
  • Enable field research with agility clubs for continuous improvement.
  • Explore native mobile app for push notifications.
